# CLAUDE.md — wasm4games

## GGEN-Only Covenant

**Never hand-edit files under `src/patterns/` or `src/chain_tests/`.**

These files are generated by `ggen` from the TTL ontology in `crates/wasm4games/ggen/`.
To change a kernel: edit the TTL/query/template, run `ggen sync`, commit the result.
Treat generated `.rs` files as first-class source — read and reason about them freely,
but repair defects upstream in the ontology, not in the generated file.

## Crate Structure

| Crate | Role | Included in Workspace |
|-------|------|-----------------------|
| `wasm4games` | Offline core — 75 branchless game-pattern kernels, `no_std` | Yes |
| `wasm4games-capi` | C-ABI bridge (`w4g_kernel(pattern_id, state, input)`) | Yes |
| `wasm4games-wasm4pm` | Online-only process-mining bridge | **EXCLUDED** |

`wasm4games-wasm4pm` is excluded from the workspace. It requires pinning specific git revs
before real use; do not add it to `[workspace]` without that step.

### wasm32 Target

```bash
cargo make wasm-check   # Compiles wasm4games --no-default-features to wasm32
```

The `wasm32-unknown-unknown` target must be installed (`rustup target add wasm32-unknown-unknown`).
Execution is unverified without a wasm runtime; `wasm-check` only confirms it compiles.

## Unsafe Code Policy

`#![forbid(unsafe_code)]` is enforced at the `wasm4games` and `wasm4games-capi` crate level.
The only audited unsafe code lives in `bcinr-logic` (upstream dependency):

| File | Blocks | Justification |
|------|--------|---------------|
| `bcinr-logic/src/mem.rs` | 1 | Memory arena bounds (proven safe) |
| `bcinr-logic/src/autonomic/packed_key_table.rs` | 1 | Type-safe byte reinterpretation |
| `bcinr-logic/src/patterns/deterministic_mpmc.rs` | 2 | Lock-free CAS (proven safe) |

Do not add unsafe blocks to wasm4games or wasm4games-capi without a full Hoare-logic proof
and peer review.

## Kernel ABI

Every pattern kernel is a pure function:

```rust
fn pattern(state: u64, input: u64) -> u64;
```

Anti-Cheat kernels (ids 71–75) return a verdict bitmask: `0` = ADMITTED, nonzero = refused.
The C-ABI surface is `w4g_kernel(pattern_id: u32, state: u64, input: u64) -> u64`.

## wasm4pm Bridge Notes

`wasm4games-wasm4pm` bridges the offline core to live process-mining infrastructure.
Before enabling it:
1. Pin the `wasm4pm` and `bcinr-logic` git revs in `Cargo.toml`
2. Add it to `[workspace.members]` explicitly
3. Verify `cargo make check` still passes with it included
