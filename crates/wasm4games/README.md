# wasm4games

A branchless, `no_std`, engine-agnostic **game-pattern foundry** built on
[`bcinr-logic`](../bcinr-logic). Game behaviors — input admission, ticks, collision, combat,
status, physics, evidence, promotion, pathfinding, economy, audio, networking, and
anti-cheat — are expressed as **bounded byte-class state** transformed by **branchless
kernels** that reuse `bcinr-logic`'s verified primitives (masks, saturating arithmetic,
DFAs, popcount, permutation networks, rolling receipts).

The catalog holds **75 patterns**: ids 1–70 across 14 families, plus an **Anti-Cheat**
family (ids 71–75). Every kernel is **generated by ggen** from the ontology under
[`ggen/`](./ggen) and committed; you regenerate kernels, you never hand-edit them.

## The kernel ABI

Every pattern kernel is a pure function over a packed-`u64` `(state, input)` pair:

```rust
fn pattern(state: u64, input: u64) -> u64;
```

Each pattern documents how it bit-packs its fields into `state` and `input`. Most kernels
return the next packed state; **Anti-Cheat detector kernels** (ids 71–75) instead return a
**verdict bitmask**: `0` = ADMITTED (legal), nonzero = refused, with each set bit naming a
refusal reason (e.g. `bit0 = teleport`). The same flat ABI is what `wasm4games-capi` exports
over the C ABI as `w4g_kernel(pattern_id, state, input)`.

## Object-centric evidence

Any meaningful transition can emit process evidence in four mirrored channels:

- `evidence::ocel` — object-centric events (an activity, a logical tick, an admission
  status, and the real objects the event links to). Allocation-free by default;
  `OcelLog` + `to_json` under `alloc`.
- `evidence::otel` — 16-bit runtime span codes (no strings on hot paths).
- `evidence::receipt` — a tamper-evident rolling receipt chain (FNV-1a, via `bcinr-logic`).
- `evidence::replay` — deterministic replay frames for re-derivable digests.

## The breed-rigor battery

Each generated kernel ships its own falsification battery, so a green test suite *means*
something:

- a branchless body **plus** a branchful `_reference` oracle, pinned by an `equivalence`
  proptest;
- three value-mutants (`cf1`/`cf2`/`cf3`) proving the suite can tell a wrong answer apart
  from the right one;
- a **two-sided fixture corpus** (`must_admit` / `must_refuse`);
- a `weakened` variant that the refuse corpus must catch (`weakened_fails_corpus`), proving
  the corpus has teeth;
- `boundaries`, a dormant `bench` module, and two Hoare-logic lines.

The whole catalog folds into a single `GOLDEN_CORPUS_DIGEST` (in `src/corpus.rs`); every
projection target (the C ABI, WASM, engine adapters) must reproduce it to claim portability.

## Doctrine

> **Engines project worlds; wasm4games operates patterns; wasm4pm admits evidence; ggen
> manufactures the law.**

wasm4games never decides admissibility itself. It *operates* patterns and *emits* candidate
evidence shaped 1:1 onto `wasm4pm-compat`. The authority that admits or refuses that
evidence is the external **wasm4pm** process miner — **referenced, not vendored**. The real
bridge lives in the workspace-**excluded**, online-only `wasm4games-wasm4pm` crate; this
crate keeps only dependency-free mirror types in `src/compat.rs`, and its in-crate `verify`
module is an offline proxy, not the authority.

## Build & test (offline-pure, no network)

```bash
# Build (default is no_std, allocation-free)
cargo build -p wasm4games

# Test (std pulls in alloc + the test harness)
cargo test -p wasm4games --features std

# Lint — warnings are errors
cargo clippy -p wasm4games --all-targets --features std,bench -- -D warnings

# wasm32 target (browser / WASM-4 fantasy console) — UNVERIFIED for the corpus digest
cargo build -p wasm4games --target wasm32-unknown-unknown --no-default-features
```

### Feature flags

| Feature   | Effect                                                    |
| --------- | -------------------------------------------------------- |
| (default) | `no_std`, zero-alloc core.                               |
| `alloc`   | Adds heap-backed types (e.g. `OcelLog`) and JSON export. |
| `std`     | Implies `alloc`; enables `std` and the test harness.     |
| `bench`   | Pulls in Criterion for benchmarks.                       |
