1//! Candidate process evidence emitted by pattern kernels.
2//!
3//! wasm4games *emits* candidate evidence; admission/refusal is the job of the external
4//! `wasm4pm` authority (see [`crate::compat`]). The types here are intentionally shaped
5//! to map 1:1 onto `wasm4pm-compat` canonical types.
6//!
7//! # Module overview
8//!
9//! | Sub-module | Purpose |
10//! |---|---|
11//! | [`ocel`] | Object-centric events (OCEL-2.0 style). Each event links to typed objects via [`ocel::ObjectRefs`]. |
12//! | [`otel`] | 16-bit runtime span codes (OpenTelemetry-inspired). Hot paths never carry strings. |
13//! | [`receipt`] | Tamper-evident rolling receipt chain (FNV-1a via `bcinr_logic`). Witnesses execution order. |
14//! | [`replay`] | Deterministic replay frames. Re-folding identical frames reproduces an identical digest. |
15//!
16//! # `no_std` compatibility
17//!
18//! All types in this module use only `core::*`. The `alloc` feature gates the heap-backed
19//! log types ([`ocel::OcelLog`]). All other types are always `const`-constructible with zero
20//! heap allocation.
21//!
22//! # Invariants
23//!
24//! * All multi-byte integers are encoded as big-endian so the byte stream is identical across
25//!   architectures — a prerequisite for cross-host receipt comparison.
26//! * OCEL events are emitted with `status` codes that mirror the [`crate::class`] admission
27//!   outcome at emission time; `wasm4pm` may re-evaluate admission later.
28//! * The receipt chain is append-only and non-cryptographic: it witnesses execution order for
29//!   replay comparison, not for adversarial tamper-proof guarantees.
30
31pub mod ocel;
32pub mod otel;
33pub mod receipt;
34pub mod replay;
