Module status 

Source
Expand description

The admission status vocabulary, as ordered u8 codes.

Ordering is meaningful: it forms a coarse “admission lattice” so a worst-of join can be computed with a branchless max. Terminal/abnormal codes (REFUSED, RESIDUAL) sort above the normal lifecycle so they dominate a join.

All constants are in [0, COUNT). COUNT itself is not a valid status code; it is the cardinality of the vocabulary for bounds-checking.
