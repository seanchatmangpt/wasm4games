Object-centric event log (OCEL-style) types.

Default build is allocation-free: an OcelEvent is Copy and links to a bounded set of objects via ObjectRefs. An alloc-gated [OcelLog] (a growable event stream) and JSON export are added on top of these core types.

§OCEL model

In OCEL-2.0, an event is an activity execution at a point in time that is associated with one or more typed objects. OcelEvent encodes exactly that: an event_code (activity type), an activity (pattern id), a timestamp (logical tick), a bounded set of (object_type_code, object_id) pairs (ObjectRefs), and the admission status code.

§Invariants

  • ObjectRefs::len() <= ObjectRefs::CAP at all times.
  • write_to is deterministic and architecture-independent (big-endian encoding).
  • An OcelEvent with no object links is valid but may indicate a laundering pattern — see ObjectRefs::is_empty.
